# Migo

Privacy-first navigation app with offline maps, ALPR avoidance, and personality-driven cartoon avatars. Built with Flutter and Supabase.

> **Core promise:** No data sold. No insurance sharing. No Google backend. Not being surveilled is a human right in this codebase.

---

## Setup

### Prerequisites

- Flutter SDK 3.4+
- Dart SDK 3.4+
- A Supabase project (free tier works)

### 1. Clone and install dependencies

```bash
git clone https://github.com/PC-Master-Race/migo.git
cd migo
flutter pub get
```

### 2. Deploy the database schema

1. Open your Supabase project → **SQL Editor → New query**
2. Paste the contents of `supabase/schema.sql` and run it
3. For development, also run `supabase/seed.sql` to get test data

### 3. Connect the backend

In `lib/services/supabase_service.dart`, fill in your project credentials:

```dart
const String supabaseProjectUrl = 'https://YOUR-PROJECT.supabase.co';
const String supabaseAnonKey    = 'YOUR-ANON-KEY';
```

The app runs in **offline mode** with placeholder credentials — map and GPS work fully without a backend.

### 4. Run the app

```bash
flutter run
```

> **Note:** Android and iOS platform folders are generated by `flutter create`. Run `flutter create --project-name migo .` once in the repo root to generate them, then add the location permission manifest entries documented in `lib/services/location_service.dart`.

## Phase 1 — What was built

### Flutter project structure
Full folder scaffold per PRODUCT_BRIEF: all models, service stubs, providers, screens, utils, widgets, and the docs landing page. Every file has a header comment. No magic numbers.

### Map rendering — three zoom modes
`flutter_map` renders OpenStreetMap tiles. `MapService` maps zoom levels to three modes (constants in `constants.dart`):

| Zoom | Mode | Tiles | Avatars |
|------|------|-------|---------|
| ≤ 13 | Cartoon | OSM + warm amber tint | Visible |
| 13–16 | Hybrid | OSM + lighter tint | Visible |
| > 16 | Street | Esri satellite | Hidden |

Attribution badges are always rendered (OSM and Esri usage policy requirement).

### GPS location service
`LocationService` wraps `geolocator`: permission flow, `bestForNavigation` stream, 5 m distance filter. Location is **optional** — the map stays on a fallback center if permission is denied. Background capability is architecturally ready; platform manifest entries are a TODO pending `flutter create` generating the platform folders.

### Follow-user camera
Map follows GPS position. Manual panning pauses follow mode and reveals a recenter button; tapping it resumes following.

### Speed HUD
Displays GPS speed in mph (jitter-suppressed below 2 mph). Shows the current road's speed limit via a throttled Overpass API query (re-queries every 120 m of movement). Degrades gracefully to "Unknown" when OSM has no data or the network is unavailable.

### Offline tile caching
**Decision: Hive CE over Isar** (documented in `map_service.dart`) — tile PNG bytes on disk, timestamps in Hive.

- `OfflineFirstTileProvider`: serves from disk, falls back to network, caches opportunistically
- `TilePrefetcher`: bulk-downloads 100-mile radius (z8–z13) on first GPS fix
  - WiFi-only by default (never auto-downloads on cellular)
  - Weekly cadence — skips if synced within 7 days
  - Politely paced at ~7 tiles/sec for the donated OSM tile servers

### Supabase schema
13 tables, all with RLS, column comments, and covering indexes. See `supabase/schema.sql`. Seed data for Bay Area dev testing in `supabase/seed.sql`.

## Design decisions log

| Decision | Options considered | Choice | Reason |
|---|---|---|---|
| Framework | React Native, Flutter, Kotlin native | Flutter | Cross-platform, strong map support, single codebase |
| Backend | Firebase, Supabase, self-hosted | Supabase | Open source, Postgres, real-time, no Google dependency |
| Map rendering | Mapbox, OsmAnd, flutter_map | flutter_map | OSM-native, free, no API key required |
| Local storage | Hive, Isar, SQLite | Hive CE | Isar maintenance stalled; Hive CE actively maintained, pure Dart |
| State management | Provider, Bloc, Riverpod | Riverpod | Modern, testable, recommended for new Flutter projects |
| Typography | Nunito, Fredoka One | Nunito | Fredoka One is display-only; Nunito has full weight range for HUD hierarchy |
| Routing engine | OSRM, Valhalla | **TBD — Phase 2** | Valhalla's dynamic costing looks promising for ALPR-avoidance penalty zones |

## What Phase 2 requires

- **Routing engine decision:** Evaluate OSRM vs Valhalla. Key criteria: offline capability, Flutter HTTP API shape, custom costing flexibility for ALPR-avoidance penalty zones and "avoid popular routes."
- **Route calculation** with all toggles: fastest (default), shortest, fuel efficient, fewest stops, avoid freeways/tolls/popular routes, ALPR avoidance.
- **Turn-by-turn instructions** displayed in the HUD.
- **Route deviation detection** and automatic recalculation.
- **Voice guidance hook** using Flutter's default TTS (ElevenLabs TODO clearly marked).

## Privacy guarantees (enforced, not aspirational)

- No Google/Meta/ad-network SDK anywhere in the codebase
- No data sold or shared with third parties
- No insurance company data sharing
- Driving session data anonymized; never linked to real identity externally
- ALPR avoidance data never sent to any third party
- Family location data: Supabase only, visible only to authorized family group members via RLS
- All Supabase tables default to deny; sharing is always explicit opt-in
